# How to persist codebase standards

Persist findings into a clean, structured document at the requested path.

## Output format

Use these headings in this order:

```markdown
# Codebase standards

## Purpose

## Programming languages

## Tooling

## Frameworks and packages

## Cloud provider and hosting

## Development methodologies

## Design patterns

## Testing

## Deployment and operations

## Important notes
```

## Rules

- Keep concrete names, versions, commands, and file paths.
- Preserve uncertainty. Use `Unknown` when evidence is missing.
- Prefer bullets for lists, short paragraphs for narrative notes.
- Do not add guessed standards. Only persist evidence from user answers or repo inspection.
- If repo has multiple stacks, separate them clearly inside relevant section.
